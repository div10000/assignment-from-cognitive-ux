Chapters

    How this works

    • Each slide is a thumbnail until clicked ("lite embed") — this avoids loading 3 YouTube iframes (and their scripts) up front, which is the biggest real-world performance cost of video carousels.
    • Only one YT.Player instance exists at a time. Switching slides destroys the old player and mounts a new one, so memory/network stay bounded regardless of carousel size.
    • Chapter timestamps are placeholders — see the README note below on why, and how to swap in real ones in under a minute.

    Automated Chapter Generator (prototype)

    Paste a YouTube URL. The tool tries to fetch the video's captions directly in the browser. That call is blocked by YouTube's CORS policy on most production domains (it's the same reason a plain client cannot read most YouTube data without a backend) — so as a fallback, you can paste a timestamped transcript instead and it will be segmented the same way.
